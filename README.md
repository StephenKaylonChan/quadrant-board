# 每日四象限

个人工作事务管理面板:按「重要性 × 是否有期限」把每天的任务摆进四象限,支持粘贴截图、历史回看、AI 拆任务草稿确认。

项目背景见 [docs/architecture/background.md](docs/architecture/background.md):它记录了为什么这个工具要同时支持紧急排查线和重要建设线。

源码仓库:https://github.com/StephenKaylonChan/quadrant-board

## 启动 / 停止

需要先打开 Docker Desktop,然后在本目录执行:

```bash
docker compose up -d        # 启动(第一次会构建镜像、装依赖,需要几分钟)
docker compose logs -f      # 看运行日志(Ctrl+C 退出,不影响服务)
docker compose down         # 停止
```

打开浏览器访问 **http://localhost:5173** 即可使用。
后端接口文档(FastAPI 自动生成):http://localhost:8000/docs

## 日常使用

- **新建任务**:右上角「+ 新任务」,回答两个问题——重要吗(二选一)、哪天截止(今天/明天/选日期/无期限,默认今天);截图后直接 Ctrl/Cmd+V 粘贴。
- **象限规则**:上下 = 重要/不重要;左右 = 无期限/有期限。**紧急度不用打分——它就是截止日期离今天多近**。有期限的象限里,截止越近排越前(已过期标红置顶),过了一夜"明天"自动变"今天"并上浮。
- **拖拽**:同一天内拖动调顺序;拖到别的日期段的卡片中间 = 改截止日;拖去左列 = 摘掉期限;上下拖 = 切换重要性。
- **每日结转**:昨天没完成的任务,今天的面板自动还在——任务不属于某一天,每天的面板只是按日期筛出来的视图。
- **回顾历史**:顶栏左右箭头切换日期,可以还原任何一天的面板(历史面板只读不可新建)。
- **任务状态**:待办 → 进行中 → 待 Review → 待验证 → 已完成。「待 Review」表示轮到别人看,「待验证」表示已合并但还要你做真实环境验证,两者都留在面板上不归档。
- **完成任务**:点开卡片,状态切到「已完成」。它会收进该象限标题行右侧的「归档」入口(点开可看、可点进去改回待办),明天起不再出现。
- **今日同步**:右上角「今日同步」会根据当前面板生成可编辑、可复制的早会同步草稿。
- **快捷键**:AI 输入框按 Enter 拆任务;编辑弹窗按 Ctrl/Cmd + Enter 保存、Esc 关闭;确认删除时 Enter 删除、Esc 取消;今日同步弹窗按 Ctrl/Cmd + Enter 复制、Esc 关闭。
- **图片**:卡片上直接显示缩略图,**点缩略图 = 复制进剪贴板**(方便贴到聊天里);编辑弹窗里点图片进入全屏预览,**预览里右键 = 复制**,Esc 或点空白关闭。
- **AI 拆任务**:面板上方的 AI 输入框,一句话(可包含好几件事)回车,AI 拆好标题/备注/**重要性/截止日期**("周五前"会换算成具体日期)/**状态**后**直接弹出预填的编辑窗**——可以改内容、贴图片,保存才入库;多条草稿会按 1/2、2/2 依次弹。
- **卡片信息**:位置序号、状态、截止徽章(已过期红色)、备注正文(小字最多两行)、图片缩略图;悬停出现 × 可删除(有二次确认)。
- **编辑弹窗**:点弹窗外面,有未保存内容会问「保存 / 不保存 / 继续编辑」,没改动直接关闭。
- **主题**:顶栏 浅色 / 深色 / 系统 三段开关,选择会记住。

## AI 配置

大模型配置在本目录 `.env` 文件(不进 git):`LLM_BASE_URL`、`CHAT_MODEL`、`LLM_API_KEY`。
改完 `.env` 要 `docker compose up -d` 重建容器才生效;没配置时 AI 输入框自动隐藏,其余功能不受影响。

## 数据在哪

全部在 `./data` 目录:`app.db` 是 SQLite 数据库(任务),`uploads/` 是图片。
**备份 = 复制 data 目录**。删容器、重建镜像都不影响数据。

## 目录结构

```
quadrant-board/
├── docker-compose.yml      # 一条命令启动前后端
├── data/                   # 数据库 + 图片(运行时生成,要备份的就是它)
├── backend/                # FastAPI 后端
│   └── app/
│       ├── main.py         # 应用入口:组装路由、静态文件
│       ├── config.py       # 路径、上传限制等配置
│       ├── database.py     # 数据库连接和会话
│       ├── models.py       # 表结构(任务表、图片表)
│       ├── schemas.py      # 接口的输入输出格式
│       └── routers/tasks.py# 全部接口逻辑(含"每日面板"的核心查询)
└── frontend/               # React + Vite 前端
    └── src/
        ├── App.tsx                    # 顶层:日期切换 + 加载任务
        ├── api.ts                     # 所有后端请求集中在这
        └── components/
            ├── QuadrantBoard.tsx      # 四象限布局和分组规则
            ├── TaskCard.tsx           # 任务卡片和缩略图复制
            ├── TaskEditor.tsx         # 新建/编辑弹窗(粘贴图片在这)
            ├── AiQuickAdd.tsx         # AI 一句话拆任务入口
            └── Lightbox.tsx           # 图片预览和右键复制
```

## 改代码

前后端都挂载进了容器并开了热重载:**改完保存,刷新浏览器就生效**,不用重启容器。
(只有改 `requirements.txt` / `package.json` 这类依赖文件时,才需要 `docker compose up -d --build`。)

## 常见问题

- **端口被占**:改 `docker-compose.yml` 里的 `5173:5173` / `8000:8000` 左边的数字。
- **页面打不开**:先 `docker compose ps` 看两个容器是不是都在跑,再 `docker compose logs -f` 看报错。
